전역변수 static
초기화 안해도 자동으로 0으로 초기화
중괄호 밖에 있다. 해당 페이지 내에 모든 지역에서 접근 가능하다.
다만 다른 페이지에서 접근시 그 변수의 소속을 알려줘야 한다.(클래스)

static변수, static 메서드

	컴파일 하자마자 JVM이 static 필드들을 먼저 메모리에 올리고
	프로그램이 종료되면 메모리에서 해제된다.
	하나의 프로젝트 안에 모든 페이지들이 static 필드를 알고 있기 때문에
	수정 혹은 변경된 값이 다시 메모리에 올라가면서 초기화 되는 경우는 없다.(보안에 취약)

메서드


	수학
f(  x   )	=	2x+1
매개변수		결과값
x는 외부값과 함수를 연결



	Java	
메서드: 기능
주로 메서드의 이름을 동사로 씀	
f(	  x	   )	=	2x+1
함수명 매개변수,파라미터		리턴값
x는 외부값과 함수를 연결
리턴값이 있을 경우 f(x)자체를 리턴 값으로 봐야 한다.(단 리턴 값이 있을 경우)

메서드 선언
(1)
리턴타입 함수명(매개변수 자료형 매개변수1,매개변수 자료형 매개변수2,...)
{
실행할 문장;
return 리턴 값;
}
(2)
void 함수명(매개변수 자료형 매개변수1,매개변수 자료형 매개변수2,...)
{
실행할 문장;
}
(3) 실행메서드
void 함수명()
{
실행할 문장;
}

메서드를 선언할 때 생각해야 할 순서

1. 틀을 잡는다.	void(리턴값이 없다.)
void f() {}
2. 매개변수를 선택한다.
void f(int x){}
3. 리턴값을 선택한다. 
C언어 JAVA는 서로 메모리 구조가 다르다. java에서 매개변수는 method field라는 곳에 저장된다.
f는 주소를 갖는 포인터이다.
void  f(int x){
return 2x+1
}

메서드 주의 사항
1. 메서드 내부에 다른 함수를 선언할 수 없다.
2. 매개변수가 없는 메서드는 외부에서 값을 전달 받을 수 없다.
3. 리턴값이 있는 메서드는 사용시 리턴값으로 보자!
4. 함수 선언시 매개변수는 저장공간이다.
	사용시에 값을 그 자리에 넣어준다.

변수를 사용하는 이유: 반복되는 값을 쉽게 관리하기 위해서
입출력 메서드를 사용하는 이유: 개발자가 한 작업에 대해 확인하기 위해서
조건문을 사용하는 이유: 결과가 2가지 이상일 경우 선택하기 위해서
반복문을 사용하는 이유: 반복되는 문장들을 쉽게 관리하기 위해서(코드를 줄이기 위해서)
배열을 사용하는 이유: 반복되는 같은 타입의 변수들을 쉽게 관리하기 위해서
메서드를 사용하는 이유: 반복되는 작업들을 쉽게 관리하기 위해서
